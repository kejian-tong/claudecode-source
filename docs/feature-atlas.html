Hidden / Gated Feature Atlas

A stricter answer to the "hidden Claude Code features" discussion. This page separates what is clearly implemented in the source tree from what is only gated, partially inspectable, or overstated by poster-style summaries.

What the viral summary gets right

Buddy, KAIROS, Auto-Dream, Ultraplan, Ultrareview, Voice, Teleport, Chrome, Terminal Panel, Team Memory, and swarm-style coordination are all real feature families in this snapshot.

What it gets too confident about

"Daemon" and "self-hosted runner" need caveats, and counts like 500+ runtime gates are only valid if the counting method is disclosed.

Best one-line judgment

This source tree looks like a shipped core runtime surrounded by a large ring of gated, preview, enterprise, and build-variant capabilities.

Claims That Need Caveats

These are the places where the viral-summary format gets too confident, or where this snapshot does not expose enough of the implementation to speak that strongly.

Over-read

Daemon

The snapshot definitely contains assistant daemon mode, daemon workers, and cron/SDK daemon references. What it does not cleanly prove is a general always-on background service mode in the broad marketing sense.

Partially inspectable

Self-Hosted Runner

src/entrypoints/cli.tsx contains a real SELF_HOSTED_RUNNER fast path, but the imported implementation directory is missing from this snapshot. The entry surface is real; the full subsystem is not fully inspectable here.

Metric caveat

"500+ Runtime Gates"

This is only a safe claim if the counting rule is stated up front. Exact grep supports 89 unique feature symbols and 941 feature call sites, but "runtime gate" is a much fuzzier bucket.

Not a code claim

Leak / Discovery Story

Provenance and disclosure chronology are not things the source tree can prove by itself. They belong in attribution notes, not in the technical feature verdicts.
